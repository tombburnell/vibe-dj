## Background

To match Spotify tracks against a large Rekordbox collection (22k+ tracks), we need:
1. A domain model representing Rekordbox tracks with normalized fields
2. TSV parser to extract track data from `rekordbox/all-tracks.txt`
3. Normalization utilities to standardize text for matching
4. Inverted index for fast candidate generation

**Note:** Rekordbox tracks are parsed into `RekordboxTSVTrack` objects for indexing/matching purposes. When matched with Spotify tracks, they are merged into unified `LibraryTrack` entities stored in SQLite database. Rekordbox-only tracks (not matched to Spotify) can optionally be added to the database as `LibraryTrack` entities with `in_rekordbox=True`.

**Structure:**
```python
class RekordboxIndex:
    def __init__(self, tracks: list[RekordboxTSVTrack]):
        self.token_index: dict[str, set[str]] = {}  # token -> set[rb_track_id]
        self.tracks: dict[str, RekordboxTSVTrack] = {}  # rb_track_id -> track
        self.token_frequency: dict[str, int] = {}  # token -> count
        self.total_tracks: int = len(tracks)
```

**Methods:**

- `build_index(tracks: list[RekordboxTSVTrack], rare_token_threshold: float = 0.05) -> None`
  - Count token frequencies
  - Only index rare tokens (< threshold% of tracks)
  - Exclude stopwords
  - Build inverted index mapping tokens to track IDs

- `get_candidates(spotify_tokens: list[str], max_candidates: int = 100) -> list[str]`
  - Union candidate sets from token index
  - Score by token overlap count

### 6. Import Command âŹł

**File:** `src/mm.py` (New - CLI entry point)

Implement `mm import-rekordbox` command:

- Parse TSV file using `RekordboxTSVParser`
- Build inverted index using `RekordboxIndex`
- For each Rekordbox track:
  - Match against existing `LibraryTrack` entities in database
  - If match found (fuzzy match on title+artist):
    - Update `LibraryTrack` with Rekordbox metadata:
      - `in_rekordbox = True`
      - `rekordbox_file_path`
      - `rekordbox_bpm`, `rekordbox_genre`, `rekordbox_key`
      - `rekordbox_match_confidence` (from matching score)
  - If no match and `--add-unmatched` flag:
    - Create new `LibraryTrack` with `in_rekordbox=True`
- Commit transaction

**Command Options:**
- `--add-unmatched`: Add Rekordbox-only tracks to database (default: false)
- `--min-confidence <float>`: Minimum confidence to update (default: 0.6)

## Implementation Notes

- `RekordboxTSVTrack` is used for parsing/indexing only - not stored in database
- Matched Rekordbox tracks will be merged into unified `Track` entities in SQLite
- Generate `rb_track_id` using hash of `file_path` (most unique identifier)
- Handle file encoding issues (UTF-16-LE detected, support multiple encodings)
- Log warnings for skipped/invalid rows but continue processing
- Index is in-memory only - rebuilt on each matching operation
